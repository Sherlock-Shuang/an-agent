# 手把手教你如何从零开始理解 Agent (代理)

欢迎来到 Agent 学习之旅！如果你经常听到别人把 "Agent(智能体)" 吹得神乎其神，好像是一个拥有硅基生命的机器人，那么这份文档就是来帮你打破滤镜的。

Agent 没有那么神秘。看完这份文档并阅读本项目的源码，你会深刻地发现：
**Agent 其实就是一个把 “大语言模型(Brain)” + “执行工具的死循环(Loop)” + “一堆 Python 技能函数(Tools)” 结合起来的普通程序而已**。

---

## 认识大模型的能力：Function Calling (函数调用)

在 Agent 之前，大模型只能“陪聊”：你输入一句话 -> 这句话传到云端 -> 云端返回一大段话给你。它是个“残疾”的聊天机器，不能联网，不能操控你的鼠标，甚至无法知道现在是几点。

后来，厂商（如 OpenAI）推出了 **Function Calling（有时叫做 Tool Calling）** 能力。这是现代所有主流智能体的基石！

**它是怎么运作的？**
1. **你（开发者）** 先写好各种 Python 函数。例如：写一个 `def get_weather(city_name):` 乃至 `def press_mouse():`。
2. **你（开发者）** 把这些能干的函数提取出**“说明书”**（就是这段代码里字典里的 description、参数属性），跟你的提示词一起发给大模型。并在提示词里告诉它：“遇到搞不定的事情，翻这本说明书”。
3. **大模型收到信息后**，它会发现你给了说明书。当用户问“今天上海天气怎么样？”时，大模型自己知道自己查不到天气，于是它不去编造废话，而是 **返回一个特定的 JSON 对象**，里面写着：“**我要调用名为 get_weather 的函数，参数是 {'city_name': '上海'}**”。
4. **你（开发者）** 在代码（也就是 `agent.py`）里收到这个特殊的“调用请求 JSON”后，用 `if` 判断拦截住它，不把这段乱七八糟的 JSON 显示给用户看，而是。**在本地默默真正执行** 你的 python 代码 `get_weather("上海")` 得到了 `晴天 20度` 的结果。
5. **你（开发者）** 再把 `{"role": "tool", "content": "晴天 20度"}` 发回给大模型。大模型拿到这个结果继续思考：“哦！拿到天气了！”，最后它生成最终话语：“今天上海是晴天，气温20度哦” 返回给你。

这，就是 Agent 的本质。

## 这个项目是怎么把上面的理论变成代码的？

这个小巧的项目被拆成三个核心文件：

### 1. `tools.py` （打工人的技能包）
在这个文件里，你定义了 `search_web`, `read_file`, `write_file` 三个真正的 Python 功能（函数）。
除了代码之外，我们还顺带写了一份叫做 `TOOLS_DEFINITION` 的字典。这个字典就是刚才所说的**说明书**。它告诉大语言模型：“嗨！你拥有这三个工具，在某个工具的介绍里我写了你应该在啥时候用。你如果要用的话，必须按照规定的参数名传参数给我！”

### 2. `agent.py` （打工人的思考大脑，也就是 ReAct 循环）
这个文件是最**最重要的**，它揭示了你在网上听到的最高大上的词语：“**ReAct 循环 (Reason + Act)**”。
在这个文件 `run()` 那个函数里，有一个 `while True:` 的死循环。

- **Reason (思考阶段)**：拿着历史对话和工具说明书去问模型。模型说：“嗯……这个问题需要用搜索工具……”（模型返回了要求执行功能的消息）
- **Act (执行阶段)**：我们的 Python 脚本捕获到了模型的请求，从本地执行完了工具，并把结果作为对话塞进历史记录里去。

因为在一个死循环中 `while True:`，大模型会一直在这个循环里不断的：思考->调用工具->拿到结果->继续思考->如果还需要东西就继续调用工具... 直到它认为问题解决了，给出一个正常纯本文形式的回答为止 (`return response_message.content`)。

### 3. `main.py` （让用户能用的交互终端）
这个单纯就是给用户的外壳。负责接受你在黑框框里面打字（`input()`），然后带着这个问题去找 `agent.py` 的大脑计算，最后把它算出来的回答打印出来 `print()`。

## 如果你学有余力，可以怎么魔改这个项目？

既然你明白了 Agent 的原理，你可以自己创造无穷多的技能。

**第一步：在 `tools.py` 里面添加新的 Python 函数。**
比如你写一个查询系统剩余内存的代码：
```python
import psutil

def get_memory_info(a_placeholder: str=""):
    mem = psutil.virtual_memory()
    return f"老大的电脑还剩 {mem.available / (1024 ** 3):.2f} GB 内存"
```

**第二步：在 `tools.py` 的 `TOOLS_DEFINITION` 里面增加说明书。**
照葫芦画瓢把 `get_memory_info` 的名字和作用用文字写下来配在里面。

**第三步：把它加到映射表里。**
在对应的 `TOOLS_MAP` 字典里添加一条 `"get_memory_info": get_memory_info`。

**结果**：你重新运行 `python main.py`，当你对它说“看看内存还剩多少”，它就会像魔法一样，自动匹配去用这套逻辑运行，然后告诉你真实的剩余内存。

这，就是 Agent！你在这其中并不是被 AI 替代了，而是成为了指导和赋予 AI 本地“眼睛和双手”的造物主。
